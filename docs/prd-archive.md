# 立项需求存档

本文档是立项时 PRD 的原始意图存档：问题陈述、用户故事与范围外清单。
它记录项目「为什么长成这样」，不描述实现现状：实现现状以 `README.md` 与 `docs/` 为准，架构决策以 `docs/adr/` 为准，领域术语以根目录 `CONTEXT.md` 为准。
个别故事已被后续架构决策有意变更（如故事 8 的严格串行写作已改为并行首写承接规划摘要链），变更以 ADR 为准。

## 问题陈述

专业写作场景普遍存在以下问题：

1. 文章缺乏结构化论证体系，观点没有可验证的依据支撑。
2. 引用不严谨，来源不可溯源，或在文末集中堆砌而非放在观点原位。
3. 长文章章节之间逻辑割裂，前后不连贯。
4. 多轮修改迭代后，智能体上下文被污染，出现「失忆」，用户感觉智能体记不得之前的要求。
5. 智能体执行流程是黑盒，外部系统无法还原其运行拓扑，用户无法观察进度与分支走向。
6. 不同环节对模型能力要求不同，但现有方案只能全局共用一套大模型配置。

## 用户故事

1. 作为专业撰稿人，我想输入写作需求后系统自动识别文章品类并匹配模板，这样我无需手动搭建文章框架。
2. 作为专业撰稿人，当我的需求匹配不上任何模板时，我想系统自动转入自由结构模式生成大纲，这样流程不会中断报错。
3. 作为专业撰稿人，我想大纲严格沿用模板的一二级标题骨架并按我的需求实例化，这样产出符合行业规范格式。
4. 作为专业撰稿人，当模板填充变量我没有提供时，我想系统用醒目占位标记继续写作而不是追问阻塞，这样首稿能一次性产出，我在审阅时再补充。
5. 作为专业撰稿人，我想每个章节拥有明确的论点和可证伪的假说，这样写作完全有据可依。
6. 作为专业撰稿人，我想假说从六个不同角度发散生成并去重筛选，这样论证体系不会塌缩成同一观点的重复表述。
7. 作为专业撰稿人，我想所有写入正文的素材都经过检索智能体的相关性校验，这样正文不会出现不可靠信息。
8. 作为专业撰稿人，我想章节严格串行写作且自动承接上一章摘要，这样长文前后逻辑连贯不割裂。
9. 作为专业撰稿人，我想引用以角标形式标注在对应观点的段落原位，这样每个观点都能就地溯源。
10. 作为专业撰稿人，我想全文引文经过独立终审对账，这样不会出现无来源的标注或张冠李戴的引用。
11. 作为专业撰稿人，我想提交一条混合了多种诉求的修改意见后，系统自动拆解为逐章节的修订指令并区分纯改写与补充佐证，这样修改精准落到位。
12. 作为专业撰稿人，我想修改只作用于指定章节，这样不会出现全文无差别重写。
13. 作为专业撰稿人，我想迭代多轮后系统仍记得我历次提过的要求和偏好，这样我不必重复交代。
14. 作为专业撰稿人，我想修订后的章节自动重新通过引文核查，这样迭代不会绕过质量门禁。
15. 作为专业撰稿人，当引文核查多次失败时，我想系统带着警告把决定权交给我而不是无限重试，这样流程永远不会卡死。
16. 作为专业撰稿人，我想任务可以随时暂停重启，服务崩溃不丢进度，这样长文写作有安全感。
17. 作为专业撰稿人，我想回退到任意历史写作版本，这样可以撤销不满意的迭代。
18. 作为系统架构负责人，我想为每个主节点和子智能体在 `.env` 中独立配置模型名称、接口地址、密钥，这样不同环节可选用不同能力规格的模型。
19. 作为系统架构负责人，我想未单独配置的运行单元自动回落全局缺省模型配置，这样不必为每个单元填写全部参数。
20. 作为系统架构负责人，我想论点、假说的数量上限和重试次数都是配置项，这样可以按业务场景调节成本与深度。
21. 作为前端可视化开发人员，我想通过独立 SSE 通道持续收到标准化运行事件流，这样可以自主搭建动态 DAG 可视化大屏。
22. 作为前端可视化开发人员，我想每条事件携带全局唯一 ID、追踪 ID、父级事件 ID 和时间戳，这样可以自动拼接完整执行拓扑。
23. 作为前端可视化开发人员，我想事件覆盖节点启停、分支流转、循环迭代、子智能体调用、状态快照、模型配置、进度七类，这样黑盒执行过程完全透明。
24. 作为前端可视化开发人员，我想通过查询参数按事件类型订阅过滤，这样可以只接收关心的事件、控制数据量。
25. 作为外部调用系统，我想在创建任务时传入自己的会话标识并在事件流中按它过滤，这样能在不依赖本系统鉴权的前提下隔离不同用户的会话。
26. 作为技术运维人员，我想全链路 LLM 调用自动上报自建 Langfuse，这样可以审计每次调用的输入输出与耗时成本。
27. 作为后续迁移子智能体的开发人员，我想接口规范与打桩实现已就位且主流程可端到端空转，这样迁移时只需让真实现满足接口规范即可替换。

## 范围外

- 任何前端页面与可视化渲染实现。
- 鉴权、权限管理、密钥托管设施。
- 模型降级容错、多套模型配置方案切换、配置热更新。
- ulmen 压缩的默认启用与调优（实验结论已定为不启用，见 README）。
- 高并发压测与模板热更新。
