## 已知偏差（验收时确认、留待后续轮次处理）

1. 残留 1 条 word_count 违规：末章「（一）核心结论」节散文 464 字，低于 2 节动态收缩后的下限 600 字。
   生成期同样报出并经「修一次」机制交付（ADR-0004 恰好一次修订的已知行为）。
   根因是文体要求「核心结论一句话凝练全篇」与节级字数下限「待实证」阈值互相冲突，是阈值实证修订轮的首个直接证据。
2. 固定章标题被 LLM 扩写（如首章「…就业质量监测概述」而非骨架原文「监测概述与数据说明」）：章型事实由 State 携带、生成期校验不受影响；成品后验须经 `--chapter-types` 显式提供章型。
3. 首章按骨架应含四小节（监测背景、数据来源与可靠性、核心指标界定、核心评估维度），成品合并为两节呈现，四项内容均覆盖；骨架小节由模板承载、无硬校验，属措辞级出入。

## 人工对照规范源（reference/报告智能体文档类型目录范例.md「书写模式」）结论

- 实证分析语体：全文中性客观、无主观情绪词；数据断言均同句挂角标（打桩素材角标 [1]..[51]）。
- 对比叙事：含纵向比与标准比的数据叙述贯穿维度章；图表驱动，全文 40+ 表行，维度章均含数据表。
- 递进逻辑：前部章节止步于归因，对策集中于末章；对策建议逐条「明责任、定时限」（牵头/落实部门 + 完成年月）。
