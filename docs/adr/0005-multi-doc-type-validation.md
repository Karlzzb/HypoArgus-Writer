# ADR-0005：多文种校验架构与调研报告落地

状态：已接受（2026-07-20）。
适用范围：rewriter_loop 的风格指南组织与校验加载、State 与任务包契约、framework_orchestrator 的模板骨架格式、docs_templates/ 模板库与文种注册表。
与 ADR-0002 的关系：本 ADR 取代其「契约零扩展」一节中 tier/doc_type 走环境变量的决策（`REWRITER_LOOP_TIER`、`REWRITER_LOOP_DOC_TYPE` 废除）；其余迁移决策不变。
规范源：`reference/报告智能体文档类型目录范例.md`（四文种目录，本 ADR 只落地其中调研报告 P0）。

## 背景

rewriter_loop 的风格校验体系（style_guide.md + style_linter）此前只为人才培养方案一个文种建模，tier（本科/高职）作为贯穿全部规则的全局维度。
参考文档定义了四个文种（调研报告 P0、人才培养方案 P1、汇报材料 P2、学术论文 P2），各自的文风、章节结构、校验要求差异大且互斥——按人培方案规则校验调研报告必然误伤。
本次决策把校验体系重构为按文种分派的架构，并实打实落地调研报告；汇报材料与学术论文留占位。

## 决策

### 文种由 template_id 确定性锚定，显式进 State

文种不由模型自由裁量。
品类识别选中模板文件后，查文种注册表（模板文件 → {文种, 变体} 的确定性映射）得到文种与变体，由 framework_orchestrator 显式写入 State（doc_type、doc_variant 字段），全链路只读、不可中途切换。
自由结构模式（无模板命中）落入「通用公文」兑底文种。
RewriteTask 任务包相应增携 doc_type/doc_variant 字段；环境变量 `REWRITER_LOOP_TIER` 与 `REWRITER_LOOP_DOC_TYPE` 废除。
否决的替代：环境变量配置（同一部署无法混合服务多文种，与产品方向矛盾）；LLM 直接输出文种枚举（多一处模型判断点，且可能与模板选择自相矛盾）；从 template_id 每次现推不进 State（关键事实不在持久化现场，排障与历史回放不可靠）。

### tier 降级为人才培养方案文种内变体

本科/高职区分是教育部层面的真实互斥约束（学制/学位措辞、培养规格三分四分），已有 9 篇语料实证，必须保留为确定性 lint；但它是人培方案特有概念，不再作为全局维度。
模板库中两个 tier 已是独立模板文件，注册表条目直接携带变体，变体与文种同源同机制锚定。
其他文种的规则与加载接口完全不感知变体。
否决的替代：彻底删除 tier（丢失已实证的硬约束，错措辞直到人工终审才暴露）；拆成两个独立文种（两文件大量重复，与语料建模相悖）。

### 风格指南每文种一份，通用层即兑底文种文件

style_guide.md 拆为 style_guides/ 目录，每文种一个 md 文件（各自内嵌 ssot-config YAML，保持同一文件双消费、零漂移）。
「通用公文.md」既是兑底文种的风格指南，也是跨文种通用规则层（口语黑名单、编号规范、素材角标与查臆造、字数上限防注水）。
加载语义：兑底文种只加载通用层；其他文种做「通用 + 文种」两层合并——列表类规则（黑名单/正则）取并集，标量类规则（字数区间等）文种覆盖通用。
否决的替代：单文件多文种键（文件膨胀至千行级，散文部分多文种混排无法按文种提取）；各文件完整自持（四份黑名单同步维护，早晚漂移）。

### 调研报告校验落地三层

以参考文档四节定义（使用场景/核心目的/书写模式/章节目录）为规范源建规，不阻塞在语料收集上：

1. 确定性 lint：章型注册（监测概述、维度章、发现诊断、结论建议）+ 必含/禁用措辞 + 主观情绪词黑名单（「令人振奋」「深感担忧」等，复用 oral_blacklist_patterns 正则机制）。
2. 数据断言强制角标：含趋势动词加数值的断言句必须同句挂素材角标，复用 fabricated_quantitative 机制变体。
3. LLM 自审扩展：对比叙事（横向/纵向/标准比）与「数据呈现 → 现象归纳 → 归因分析 → 对策建议」四步递进逻辑等语义级约束，进 audit 裁决项。audit 裁决项与 lint 规则同源同机制——裁决项定义进各文种风格指南的 ssot-config，按「通用 + 文种」两层合并加载，通用层只保跨文种项；不出现「lint 按文种、audit 全局」的半拆分。

字数控制自声明三级区间，初值参照飞书范例体量拍定，ssot-config 注释显式标记「待实证」；维度章标 table_required 享受既有表章豁免。

调研报告的量化断言校验收紧为「必须同句挂素材角标」——references 数值匹配不再豁免（区别于通用层量化查臆造的「角标或 references 依据二选一」）。
调研报告的公信力卖点是数据可溯源，溯源必须在正文可见而非藏在校验器内部；这是「文种覆盖通用」标量/开关类合并语义的首个真实用例。
递进逻辑以参考文档为准的四步为准：数据呈现 → 现象归纳 → 归因分析 → 对策建议（早期表述漏掉「现象归纳」，予以更正）。
对策建议每条「明责任、定时限」（牵头部门 + 完成时限）升为确定性 lint 的第一层规则；维度章观点标题、亮点 3-5/短板 1-3、核心结论一句话句式等语义级或数量级软约束不硬 lint，进 LLM 自审裁决项或留 golden 验收轮实证后定。

### 模板骨架支持可重复章

调研报告的维度章数量随选题不定，模板骨架格式扩展可重复章标记（repeat: 1..N），实例化时 LLM 按用户需求展开为 N 个具体维度章；首尾固定章仍按序号强制对齐，章序号连续性由既有编号校验兜底。
调研报告模板骨架内首章「监测概述与数据说明」固定含四个小节（监测背景、数据来源与可靠性、核心指标界定、核心评估维度），由骨架承载，不新增校验。
docs_templates/ 新建调研报告模板并登记 index.md；现有四模板全部登记注册表：本科/高职模版 → 人培方案两变体，总结（汇报）模版 → 汇报材料（本次只挂通用规则占位），学院级多专业 → 人培方案。

模板 repeat 表达力三条封顶（防滑向模板 DSL）：① 只允许标在章级（H2），不允许小节级重复、不允许嵌套；② 一份模板至多一个可重复章位，多个重复位的需求出现时须新开 ADR 论证；③ 不引入条件章、章间引用等任何其他结构化标记——模板骨架保持「固定章序列 + 至多一个重复位」的表达力。原设计只用一个重复位，此处将隐含边界显式化。

### 维度章章型由 State 携带，不做推断

维度章标题是自由的「观点标题」（如「空间下沉：精准破局『基层人才荒』」），无法靠标题匹配或 aliases 归并识别章型。
framework_orchestrator 实例化可重复章时是唯一确知每章身份的时刻，将「章序号 → 章型」映射作为骨架事实写进 State（任务包契约一次性扩展），lint 直接消费，不从位置或标题反推。
与文种由 template_id 确定性锚定同理：关键事实在确知时刻固化下来，比事后推断稳健，且 lint 侧少一套推断逻辑。
否决的替代：位置推断（依赖首尾章标题先被正确识别，中间混入非维度章会误判）；标题模式匹配（强约束标题格式，与「观点标题须凝练便于传播」冲突）。

### 加载维度封顶为「文种 + 变体」二元

规则加载的维度严格锁定为文种 + 变体二元。
现有指南中的 `domain`（金融强国/以义取利等金融领域专属政治语）不升为第三个全局加载参数，降级为人才培养方案文种内部的规则级条件标签，与 tier 同等待遇——不进 State、不进注册表、不进加载接口，只是人培风格指南 ssot-config 里个别规则自带的条件标签，其他文种完全不感知。
任何比「文种 + 变体」更细的条件都只能是规则内部标签，不得升维为加载参数。
否决的替代：把 domain 做成第三个全局加载参数（隐性全局维度残留，正是要避免的深层级逻辑来源）。

### 通用层准入原则

「通用公文」层的准入标准 = 对全部四文种无争议成立；有一个文种存疑就不入，下沉到文种层。
据此：口语黑名单（我们/咱们/接下来等）、中文数字编号规范、素材角标语义、查臆造、字数三级完整区间（含下限，兑底文种同受约束，防注水与防写少）入通用层。
学术断言句式禁令（「XX 是 XX 的必要条件」「XX 正向预测 XX」）下沉到文种层——学术论文文种（P2，将来落地）恰要用实证研究句式，此禁令对它反向伤害；人培与调研报告两个文种文件各持一份。
接受这条正则在两处的小重复，换取列表类规则合并语义永保纯并集、永不引入「文种豁免/剔除」机制（那才是真正的深层级逻辑）。
标量类规则（字数区间等）文种需不同体量时靠「文种覆盖通用」语义换自己的区间，同样不需要豁免机制。

### 提示词瘦身纪律

拆分后加载语义是「通用散文 + 文种散文」两层注入 writer system prompt，约束条数膨胀会拉低 LLM 对超长约束清单的遵从率，是产出质量的首要威胁（lint 侧是确定性代码，复杂度只影响维护成本）。
两条纪律：① 凡确定性 lint 能兜底的规则，散文侧不重复展开（最多一句提示）——现有 do/don't 清单、boilerplate 句库多已由 lint 强制，拆分时顺手从散文剥离；② 单次注入设软预算（通用散文 + 文种散文合计不超过现指南散文体量，约 120 行不含 YAML），超预算作 review 检查项而非运行时强制。

### 验收

Fake 单测覆盖新规则逐条正/反例、可重复章实例化、注册表映射与两层合并语义。
真实链路按五维机检跑通一篇调研报告，人工对照参考文档验收；验收稿存入仓库作为调研报告首个 golden 语料，后续据此回头实证修订「待实证」阈值。
五维机检的五个维度框架文种无关、保持不变，但脚本消费的规则来源改为按 doc_type 复用 lint 新接口：`scripts/verify_article.py` 参数从 `--tier` 迁为 `--doc-type/--doc-variant`（lint 接口从按 tier 改为按文种+变体的强制随动）；验收基准按文种分目录，人培基准不动，调研报告新增自己的基准。

## 后果

- 汇报材料与学术论文暂无专属规则，命中其模板时只有通用层 + 提示词约束，质量门禁弱于 P0/P1 文种；后续落地时只需加风格指南文件与注册表登记，架构不再动。
- 调研报告字数区间等阈值在首个 golden 语料产出前是拍定值，可能误伤或漏放，需在验收轮修订。
- ADR-0002 的「风格指南随包携带不可配」原则不变，只是单文件变目录。
