# ADR-0005：多文种校验架构与调研报告落地

状态：已接受（2026-07-20）。
适用范围：rewriter_loop 的风格指南组织与校验加载、State 与任务包契约、framework_orchestrator 的模板骨架格式、docs_templates/ 模板库与文种注册表。
取代 ADR-0002 中 tier/doc_type 走环境变量的决策（`REWRITER_LOOP_TIER`、`REWRITER_LOOP_DOC_TYPE` 废除）。
规范源：`reference/报告智能体文档类型目录范例.md`（四文种：调研报告 P0、人才培养方案 P1、汇报材料 P2、学术论文 P2；本 ADR 落地调研报告，汇报材料与学术论文留占位）。

## 文种由 template_id 确定性锚定，显式进 State

文种不由模型自由裁量。
品类识别选中模板文件后，查文种注册表（模板文件 → {文种, 变体} 的确定性映射）得到文种与变体，由 framework_orchestrator 显式写入 State（doc_type、doc_variant 字段），全链路只读、不可中途切换。
自由结构模式（无模板命中）落入「通用公文」兜底文种。
RewriteTask 任务包相应增携 doc_type/doc_variant 字段。
否决：环境变量配置（无法混合服务多文种）；LLM 输出文种枚举（多一处模型判断点且可能与模板选择矛盾）；每次从 template_id 现推不进 State（关键事实不在持久化现场，排障不可靠）。

## tier 降级为人才培养方案文种内变体

本科/高职区分是真实互斥约束（学制/学位措辞、培养规格三分四分），保留为确定性 lint；但它是人培方案特有概念，不再作为全局维度。
注册表条目直接携带变体，变体与文种同源同机制锚定；其他文种的规则与加载接口完全不感知变体。

## 风格指南每文种一份，通用层即兜底文种文件

style_guide.md 拆为 style_guides/ 目录，每文种一个 md 文件（各自内嵌 ssot-config YAML，同一文件双消费、零漂移）。
「通用公文.md」既是兜底文种的风格指南，也是跨文种通用规则层。
加载语义：兜底文种只加载通用层；其他文种做「通用 + 文种」两层合并——列表类规则（黑名单/正则）取并集，标量类规则（字数区间等）文种覆盖通用。
否决：单文件多文种键（膨胀至千行级、散文无法按文种提取）；各文件完整自持（多份黑名单早晚漂移）。

## 通用层准入原则

「通用公文」层的准入标准 = 对全部四文种无争议成立；有一个文种存疑就不入，下沉到文种层。
据此入通用层：口语黑名单、中文数字编号规范、素材角标语义、查臆造、字数三级完整区间。
学术断言句式禁令下沉到文种层（学术论文文种恰要用实证句式），人培与调研报告各持一份——接受小重复，换取列表类合并语义永保纯并集、永不引入「文种豁免/剔除」机制。

## 加载维度封顶为「文种 + 变体」二元

规则加载的维度严格锁定为文种 + 变体二元。
任何更细的条件（如 `domain` 金融领域政治语）只能是规则内部的条件标签——不进 State、不进注册表、不进加载接口，不得升维为加载参数。

## 调研报告校验三层

1. 确定性 lint：章型注册（监测概述、维度章、发现诊断、结论建议）+ 必含/禁用措辞 + 主观情绪词黑名单（复用 oral_blacklist_patterns 机制）；对策建议每条「明责任、定时限」（牵头部门 + 完成时限）。
2. 数据断言强制角标：含趋势动词加数值的断言句必须同句挂素材角标（收紧为不接受 references 依据豁免——调研报告的公信力卖点是数据可溯源，溯源必须在正文可见）。
3. LLM 自审：对比叙事与「数据呈现 → 现象归纳 → 归因分析 → 对策建议」四步递进等语义级约束进 audit 裁决项；裁决项定义与 lint 同源同机制，进各文种 ssot-config 按两层合并加载。

字数三级区间初值参照范例体量拍定，ssot-config 注释标记「待实证」；维度章标 table_required 享受表章豁免。

## 模板骨架支持可重复章

调研报告的维度章数量随选题不定，模板骨架格式扩展可重复章标记（repeat: 1..N），实例化时 LLM 按用户需求展开；首尾固定章按序号强制对齐。
repeat 表达力三条封顶（防滑向模板 DSL）：只允许章级（H2）、一份模板至多一个重复位、不引入条件章或章间引用等其他结构化标记。

## 维度章章型由 State 携带，不做推断

维度章标题是自由的「观点标题」，无法靠标题匹配识别章型。
framework_orchestrator 实例化可重复章时将「章序号 → 章型」映射作为骨架事实写进 State，lint 直接消费，不从位置或标题反推。
否决：位置推断（中间混入非维度章会误判）；标题模式匹配（与「观点标题须凝练」冲突）。

## 提示词瘦身纪律

加载语义是「通用散文 + 文种散文」两层注入 writer system prompt，约束条数膨胀会拉低遵从率，是产出质量的首要威胁。
两条纪律：凡确定性 lint 能兜底的规则，散文侧不重复展开（最多一句提示）；单次注入设软预算（合计约 120 行不含 YAML），超预算作 review 检查项。

## golden 语料与五维机检按文种分派

真实链路验收稿存入 `docs/golden/<文种>/` 作为该文种的 golden 语料（「待实证」阈值的实证依据与回归对照），运行时不被任何代码读取。
`scripts/verify_article.py` 参数为 `--doc-type/--doc-variant`，验收基准按文种分目录。
