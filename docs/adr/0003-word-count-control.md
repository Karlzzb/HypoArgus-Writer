# ADR-0003：rewriter_loop 字数管控口径

状态：已接受（2026-07-20）。
适用范围：`agents/rewriter_loop/` 的字数统计、字数区间校验、写作提示词字数目标注入与修后字数复检。
与 ADR-0002 的关系：本 ADR 是其「修一次、修后不复检」设计的唯一例外记录（仅限字数复检），其余双层校验分工不变。

## 背景

真实现落地后（GitHub issue #8），产出正文暴露两类质量问题（GitHub issue #15/#16）：
章节体量失控——有的章寥寥数百字只列结论，有的章注水数千字堆砌套话；
模型对「充分论证」的优先级感知弱——系统指令里机械规则枚举喧宾夺主，论证要求淹没在禁令清单中。
确定性校验器此前没有任何字数规则，写作提示词也不携带目标字数，模型无从对齐体量预期。

## 决策

### 三级区间与最终硬标准

字数区间分三级：章（`## 一、`）2000～5000 字、节（`### （一）`）600～1500 字、小节（`#### 1.`）200～500 字。
章区间是最终硬标准；节/小节区间按同级数量动态收缩后再校验，避免多节章按静态区间累加必然超章上限的自相矛盾。
收缩公式：上限取「配置上限」与「章上限÷同级数量」的较小值；下限取「配置下限×0.7」与「章下限÷同级数量」的较大值；收缩后保证下限不高于上限。

### 散文统计口径

字数只统计散文：排除 markdown 表格（含列头、分隔行、数据行）、素材角标、行内与块级公式、围栏代码块、参考文献与附录；标题行计入。
计数规则：汉字按字计，连续半角字母数字为一词，标点与空白一律不计。
排除表格是口径的关键取舍：表章的信息由表承载，若表内容计入字数会把「补表」误奖励为「凑字」。

### 节级同级差异

同级平衡只查章内节级：最长节不超过最短节的 2 倍。
不查小节级同级差异，也不做跨章比对——前者噪声过大（小节体量本就允许悬殊），后者属全局校验层职责（另见 GitHub issue #18）。

### 表章豁免

表章（`table_required` 章型）豁免各级散文下限与节级同级差异比对，仅保各级上限防注水。
理由：表章信息密度在表内，散文只做引言与解读，压散文下限会诱导模型在表外堆砌叙述段凑字。

### 配置单一事实源

全部区间数值与系数收在风格指南 `style_guide.md` 的 ssot-config YAML 块 `word_count` 节。
校验器、修后复检、写作提示词字数目标块三方消费同一配置，零漂移；无 `word_count` 配置或正文无标准章结构时整体不校验。

### 提示词目标注入与 prompt 再平衡

draft/revise 的上下文块动态携带本章目标字数区间：叙述章型提示取区间中上限并要求同级均衡；表章提示取中下限并明令不得表外堆砌凑字。
系统指令同批再平衡：论证指令（充分论证每个论点、结合假说展开、给依据与技术路径）置于最前；删除机械规则枚举行（linter 兜底、对模型无用）；禁令压缩收尾，空泛总结词禁令补充「显著提升」「有效解决」等无据定性断言词。

### 修一次后字数复检（「修后不复检」的唯一例外）

修前检出过字数违规时，修订产物用纯函数复检字数，结论无论达标与否均如实折入 `self_check.issues`（前缀 `[word_count_recheck]`），并发 `word_count_recheck` 进度事件。
只复检字数不复检其他规则：字数复检是零 LLM 成本的纯函数统计，且字数是修订最容易复发的违规；引用类等其余规则维持修后不复检、由全局终审兜底的既有分工。
复检不触发二次修订——修订仍恰好一次，复检只保证结论对终审与人工如实可见。

## 后果

- 章节体量获得确定性底线：超上限、不足下限、节级失衡均以 `word_count` 规则进违规链并触发修一次。
- 提示词与校验器同源同口径，调区间只改 ssot-config 一处。
- 动态收缩使区间对 2～8 节的常见章结构自适应，但极端结构（如单章十余节）收缩后的下限可能形同虚设，属已知边界。
- `self_check.issues` 语义扩充：除修前结论外可含修后字数复检结论，消费方（终审、人工审核）按 `[word_count_recheck]` 前缀区分。
- 字数达标不等于论证充分——字数是论证质量的必要非充分代理指标，量化断言查臆造等语义校验另行推进（GitHub issue #17）。
