# ADR-0003：rewriter_loop 字数管控口径

状态：已接受（2026-07-20）；字数单项复检（`[word_count_recheck]`）已被 ADR-0004 的修后全量 lint 复检取代，三级区间、动态收缩与提示词字数目标注入仍以本 ADR 为准。
适用范围：`agents/rewriter_loop/` 的字数统计、字数区间校验与写作提示词字数目标注入。

## 三级区间与最终硬标准

字数区间分三级：章（`## 一、`）2000～5000 字、节（`### （一）`）600～1500 字、小节（`#### 1.`）200～500 字。
章区间是最终硬标准；节/小节区间按同级数量动态收缩后再校验，避免多节章按静态区间累加必然超章上限的自相矛盾。
收缩公式：上限取「配置上限」与「父级上限÷同级数量」的较小值；下限取「配置下限×0.7」与「父级下限÷同级数量」的较大值；收缩后保证下限不高于上限。
父级区间取直接上级：节收缩以章区间为父级，小节收缩以节区间为父级。
极端结构（如单章十余节）收缩后的下限可能形同虚设，属已知边界。

## 散文统计口径

字数只统计散文：排除 markdown 表格、素材角标、行内与块级公式、围栏代码块、参考文献与附录；标题行计入。
计数规则：汉字按字计，连续半角字母数字为一词，标点与空白一律不计。
排除表格是口径的关键取舍：表章的信息由表承载，若表内容计入字数会把「补表」误奖励为「凑字」。

## 节级同级差异

同级平衡只查章内节级：最长节不超过最短节的 2 倍。
不查小节级同级差异（噪声过大），不做跨章比对（属全局校验层职责）。

## 表章豁免

表章（`table_required` 章型）豁免各级散文下限与节级同级差异比对，仅保各级上限防注水。
理由：表章信息密度在表内，压散文下限会诱导模型在表外堆砌叙述段凑字。

## 配置单一事实源

全部区间数值与系数收在风格指南 ssot-config YAML 块的 `word_count` 节。
校验器与写作提示词字数目标块消费同一配置，零漂移；无 `word_count` 配置或正文无标准章结构时整体不校验。

## 提示词目标注入

draft/revise 的上下文块动态携带本章目标字数区间：叙述章型提示取区间中上限并要求同级均衡；表章提示取中下限并明令不得表外堆砌凑字。
系统指令以论证指令置前（充分论证每个论点、结合假说展开、给依据与技术路径），机械规则枚举不进提示词（linter 兜底），禁令压缩收尾。
字数达标不等于论证充分——字数是论证质量的必要非充分代理指标。
