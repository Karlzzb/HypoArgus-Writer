# ADR-0004：修后复检 v2 与 revise/fix 合并

状态：已接受（2026-07-20）；决策 1 的「折叠在 rewriter 编排体内」已被 ADR-0006 部分取代（折叠职责迁入章级评审），决策 2、3 不变。
适用范围：`agents/rewriter_loop` 写作编排与其 self_check 契约的下游消费方。
动因：真实 E2E 分账确认「修后不复检（v1）」会把已修好的章反复打回（`citations_ok` 钉死在修前结论），revise 模式的修一次是第二次全章写作调用且丢弃第一次产物，两者均属设计缺陷。

## 1. self_check 折叠修后终态（修后复检 v2）

修一次之后对修订产物做终态质检：全量 lint（纯函数、零 LLM 成本）+ 一次轻量自审（仅素材池非空时）。
`self_check.issues` 只留终态仍存的违规与退化说明；`citations_ok` 据终态判定：终态无引用类违规即 True。
修前违规的规模与环节仍经 SUBAGENT_PROGRESS 事件对外可观测（ADR-0001 约束 2）。
下游把 issues 视为未决问题清单即可，无需区分修前/修后残迹。

## 2. revise 与 fix 合并（revise 即修）

revise 模式先对现有正文做预 lint（纯函数），把既存违规连同修订指令并入唯一一次 revise 调用；提示词声明修复违规优先于「保持原样」但不得扩大改动范围。
调用后 lint + 自审即为终态质检，不再触发第二次写作调用；终态仍存的违规如实折叠，交全局终审在重试预算内裁决——收敛边界与 ADR-0003 一致。

## 3. 引文门禁规则前移进写作提示词

draft/revise 系统提示词携带门禁同口径的三条硬规则：量化断言同句挂角标且数值有据；角标所在句如实转述素材关键数值与对比结论、不得弱化；照抄或改写自素材的句子必须挂角标。
字数目标块追加「机检硬标准」声明，从源头减少回退轮的发生。
