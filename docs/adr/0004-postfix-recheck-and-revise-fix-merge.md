# ADR-0004：修后复检 v2 与 revise/fix 合并

状态：已接受（2026-07-20，issue #20）。
适用范围：`agents/rewriter_loop` 写作编排（writer.make_writer_run）与其 self_check 契约的全部下游消费方（citation_validator 步骤 3、demo 摘要推导）。
与既有 ADR 的关系：ADR-0003 的有界收敛设计（修一次 + 引文门禁 2 次重试）不变；本 ADR 只改单章内的复检口径与 revise 链路形态，并以全量 lint 复检取代 ADR-0003 的字数单项复检。

## 背景

issue #20 对真实 E2E（约 19.6 分钟、61 次串行 LLM 调用）分账后确认：引文门禁两轮重试合计约 9 分钟、占近一半时长。
放大机制有两个，均属设计缺陷而非模型能力问题。

其一，修后不复检（v1）把 `citations_ok` 钉死在修前结论：单章只要修前检出过任何引用类违规，即便修一次已真正规避，`citations_ok` 仍为 False；门禁把它折为 self_check_failed、整章打回重写，重写后同样机制再次触发，形成「修好了也要返工」的空转。
首轮 7/8 章不合格的大头即来源于此。

其二，revise 模式的修一次是第二次全章写作调用（约 28 秒），且其输入仍是旧正文——第一次 revise 的产物被整体丢弃，属纯浪费。

## 决策

### 1. self_check 折叠修后终态（修后复检 v2）

draft 模式修一次之后，对修订产物做终态质检：全量 lint（纯函数、零 LLM 成本）+ 一次轻量自审（约 1.3 秒，仅素材池非空时）。
`self_check.issues` 只留终态仍存的违规与退化说明；已被修一次真正规避的违规不再残留。
`citations_ok` 据终态判定：终态无引用类违规即 True。
修前违规的规模与环节仍经 SUBAGENT_PROGRESS 事件（lint_done / audit_done / revise_triggered 计数）对外可观测，不因契约收紧而丢失轨迹（ADR-0001 约束 2）。

### 2. revise 与 fix 合并（revise 即修）

revise 模式先对现有正文做预 lint（纯函数），把既存违规连同修订指令并入唯一一次 revise 调用；提示词声明修复违规优先于「保持原样」但不得扩大改动范围。
调用后 lint + 自审即为终态质检，不再触发第二次写作调用；终态仍存的违规如实折叠，交全局终审在重试预算内裁决。

### 3. 引文门禁规则前移进写作提示词

draft/revise 系统提示词携带门禁同口径的三条硬规则：量化断言同句挂角标且数值有据；角标所在句如实转述素材关键数值与对比结论、不得弱化；照抄或改写自素材的句子必须挂角标。
字数目标块追加「机检硬标准」声明。
目标是把首轮门禁不合格率从 7/8 降到偶发，从源头减少回退轮的发生。

## 后果

- 每个触发修一次的 draft 章多付一次轻量自审（约 1.3 秒），换取门禁不再把已修好的章打回（单章回退成本约 56 秒）；revise 模式单章从两次写作调用降为一次。
- self_check 契约语义收紧为「终态结论」：下游把 issues 视为未决问题清单即可，无需再区分修前/修后残迹；`[word_count_recheck]` 前缀与 `recheck_word_count` 单项复检随之删除。
- revise 单次调用须同时满足定向修订、违规规避与字数区间；若产物仍违规，本轮不补救，依赖门禁的有界重试兜底——收敛边界与 ADR-0003 一致。
